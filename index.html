List Comprehensions

Convenient way to create a new list based on the values of an existing list

[ expression for element in iterable ]



>>>	[ x * x  for x in range(10) ]
[0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

>>>	[ [x, pow(x, 3) ]  for x in range(5) ]
[[0, 0], [1, 1], [2, 8], [3, 27], [4, 64]]

[ x * x  for x in range(10) if x % 3 == 0 ]
[0, 9, 36, 81]

Mutable vs Immutable

a mutable object can be changed after it is created, and an immutable object can’t

Mutable objects:
list, dict, set, byte array

Immutable objects:
int, float, complex, string, tuple, frozen set, bytes

https://medium.com/@meghamohan/mutable-and-immutable-side-of-python-c2145cf72747
Mutable vs Immutable

- Mutable vs immutable type of variable becomes important

- When dealing with functions (passing in mutable variables) and optimizing code for efficiency

- Use mutable objects when the size is going to change, e.g. lists, dictionaries, etc.

- Use immutable objects when the value should remain the same throughout the program

- Note: the “value” of an immutable object can’t change, but it’s constituent objects can
