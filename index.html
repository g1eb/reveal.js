Databases
  • When you have lots of data (i. e., too much to store in main memory), you need to:
    • organize your data
    • be able to find what you need quickly

  • A database is a logically coherent collection of data with inherent meaning
  • Represents some aspect of the real world
  • Built for a specific purpose
  • Maintains metadata (data about data): describes data characteristics and relationships

Database Administrator
  • A database administrator (DBA) is a person responsible for the design, implementation, maintenance, and repair of an organization‚Äôs database.
  • The role includes the development and design of database strategies, monitoring and improving database performance and capacity, and planning for future expansion requirements.

SQL Databases

Different SQL Databases
  • MySQL: One of the most popular open-source relational database management systems.
  • PostgreSQL: Powerful, open-source object-relational database system that uses and extends the SQL language
  • SQLite: A C-language library that implements a small, fast, self-contained, high-reliability, full-featured SQL database engine.
  • Oracle: A enterprise-level database management system produced and marketed by Oracle Corporation.
Relational Databases

Relational databases model data by storing rows and columns in tables.

The power of the relational database lies in its ability to efficiently retrieve data from those tables (in particular when a query spans multiple tables)

https://en.wikipedia.org/wiki/Relational_database
Power of Databases
  • Can handle huge tables
    • Tables are a bit like spreadsheets
  • Can join tables. Example:
    • Table 1: Students with student ID, name, email address
    • Table 2: Courses with course ID, full name, short form, location
    • Table 3: Enrollments with student ID and course ID
    • Query: List the names of students enrolled in DSCI 510
  • Can define transactions
    • Example: transferring money from one bank account to another
    • Horror scenario: debit money ‚Üí crash ‚Üí failed to credit money
    • Begin transaction ‚Üí action ‚Üí action ‚Üí commit transaction‚Ä¢
  • Can do all this and more very quickly and efficiently
Terminology
  • Database ‚Äì contains many tables
  • Relation (or table) ‚Äì contains tuples and attributes
  • Tuple (or row) ‚Äì a set of fields that generally represent an "object" (entity) like a person or a music track
  • Attribute (or column) ‚Äì one of possibly many elements of data corresponding to the object represented by the row

Database Management System (DBMS)

Collection of programs that:

  • Manage the database structure
  • Evaluate queries efficiently against the data
  • Control access to data stored in the database
  • Manage concurrent access/modifications by multiple users

SQL Database Example

Example of a SQLite database with two tables:


CREATE TABLE Users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    email TEXT NOT NULL UNIQUE
);

CREATE TABLE Orders (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    product_name TEXT NOT NULL,
    quantity INTEGER NOT NULL,
    FOREIGN KEY (user_id) REFERENCES Users(id)
);

Database Normalization
  • There is *tons* of database theory - way too much to understand without excessive predicate calculus
  • Do not replicate data - reference data - point at data
  • Use integers for keys and for references
  • Add a special "key" column to each table which we will make references to. By convention, many programmers call this column "id"

NoSQL Databases

NoSQL Databases
  • popular for their scalability, performance, and flexibility
  • can handle large volumes of unstructured, semi-structured, or structured data
  • variety of data models:
    • documents
    • graphs/trees
    • key-value stores
    • wide-column stores
NoSQL Databases
  • MongoDB: document-oriented database that stores data in JSON-like formats
  • Cassandra: highly scalable, high-performance distributed database
  • Elasticsearch: capable of complex search operations
  • Redis: in-memory key-value store, used for caching and queueing messages
  • Neo4J: a graph database
